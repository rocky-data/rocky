# rocky-vscode

VS Code extension for the Rocky SQL transformation engine. LSP client that communicates with `rocky lsp` over stdio, plus commands for AI features, diagnostics, and operational tasks.

## Claude Code skills

VS Code-local skills live in `.claude/skills/` and activate when working here:

- [`tailwind-plus-elements`](.claude/skills/tailwind-plus-elements/SKILL.md) — using `@tailwindplus/elements` headless web components (dialog, command palette, autocomplete/select, dropdown/popover, disclosure, tabs) in the React webviews under `webview-ui/`. Read it before building or upgrading an interactive webview surface.

## Architecture

```
Extension (TypeScript)  ──stdio JSON-RPC──▶  rocky lsp (Rust binary)
```

The extension is an LSP client. All language intelligence (diagnostics, completion, hover, go-to-definition, rename, etc.) comes from the Rocky CLI's built-in LSP server. The extension handles:
- LSP client lifecycle (start/stop/restart) plus an `LspState` lifecycle emitter consumed by sidebar views
- Status bar with server state + error count
- 58 commands grouped by concern under `src/commands/` (AI, branch, compile, hooks, info, inspect, lineage, migration, ops, preview, run, storage, test, plus the `commandPalette` entry point and 2 inline code-lens commands)
- Code lens (inline run/compile above models)
- Activity bar sidebar with 6 views (Get Started, Extension Info, Models, Runs, Sources, Help) — empty-state copy is gated on the `rocky.hasProject` context flag so a workspace without a `rocky.toml` shows orientation instead of CLI errors
- Lineage webview (DOT → SVG via viz.js)
- Test explorer integration
- Drift diagnostics, formatting, folding providers

## Coding Standards

### TypeScript
- Target: ES2022, Module: CommonJS
- Strict mode enabled
- VS Code API minimum: 1.120.0 (must match the version pinned in `src/test/runTest.ts`; test runner activates against that exact build)
- Use `cp.execFile()` for subprocess calls (not `cp.exec()` — no shell injection)
- Use `vscode.window.withProgress()` for long-running commands
- Escape HTML in webview content via `escapeHtml()`

### Dependencies
- Runtime: `vscode-languageclient ^9.0.1`
- Dev: `typescript ^6.0.2`, `@vscode/vsce ^3.0.0`, `mocha ^11.7.5`
