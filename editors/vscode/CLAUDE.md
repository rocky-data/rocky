# rocky-vscode

VS Code extension for the Rocky SQL transformation engine. LSP client that communicates with `rocky lsp` over stdio, plus commands for AI features, diagnostics, and operational tasks.

## Architecture

```
Extension (TypeScript)  ──stdio JSON-RPC──▶  rocky lsp (Rust binary)
```

The extension is an LSP client. All language intelligence (diagnostics, completion, hover, go-to-definition, rename, etc.) comes from the Rocky CLI's built-in LSP server. The extension handles:
- LSP client lifecycle (start/stop/restart)
- Status bar with server state + error count
- 30 commands grouped by concern under `src/commands/` (AI, compile, hooks, inspect, lineage, migration, ops, run, storage, test)
- Code lens (inline run/compile above models)
- Activity bar sidebar with 3 views (models, runs, sources)
- Lineage webview (DOT → SVG via viz.js)
- Test explorer integration
- Drift diagnostics, formatting, folding providers

## Coding Standards

### TypeScript
- Target: ES2022, Module: CommonJS
- Strict mode enabled
- VS Code API minimum: 1.110.0
- Use `cp.execFile()` for subprocess calls (not `cp.exec()` — no shell injection)
- Use `vscode.window.withProgress()` for long-running commands
- Escape HTML in webview content via `escapeHtml()`

### Dependencies
- Runtime: `vscode-languageclient ^9.0.1`
- Dev: `typescript ^6.0.2`, `@vscode/vsce ^3.0.0`, `mocha ^11.7.5`

## Common Commands

## File Associations

- **Language ID**: `rocky` (`.rocky` files)
- **Document Selector**: `.rocky` files + `.sql` files under `**/models/**`
- **File Watchers**: `**/*.rocky`, `**/*.toml`, `**/models/**/*.sql`
