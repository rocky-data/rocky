# rocky-vscode вЂ” Agent Instructions

VS Code extension for the Rocky SQL transformation engine. LSP client that communicates with `rocky lsp` over stdio, plus commands for AI features, diagnostics, and operational tasks.

## Architecture

```
Extension (TypeScript)  в”Ђв”Ђstdio JSON-RPCв”Ђв”Ђв–¶  rocky lsp (Rust binary)
```

The extension is an LSP client. All language intelligence (diagnostics, completion, hover, go-to-definition, rename, etc.) comes from the Rocky CLI's built-in LSP server. The extension handles:
- LSP client lifecycle (start/stop/restart) вЂ” `lspClient.ts`
- Status bar with server state + error count вЂ” `extensionState.ts`
- Commands grouped by concern under `src/commands/`
- Activity-bar tree views (models, sources, runs, schema, previews, branches, вЂ¦) вЂ” `src/views/`
- The Rocky Inspector вЂ” a bottom-panel React + ReactFlow webview under `webview-ui/` (per-model trust dashboard + lineage canvas)
- Legacy plain-HTML webviews (run summary, doctor) вЂ” `src/webviews/`
- Code lens, task provider, test explorer вЂ” top-level `src/` files
- Type-safe CLI output via generated TypeScript interfaces вЂ” `src/types/generated/`

## Coding Standards

- Target: ES2022, Module: CommonJS, strict mode
- VS Code API minimum: 1.120.0
- Use `cp.execFile()` for subprocess calls (not `cp.exec()` вЂ” no shell injection)
- Use `vscode.window.withProgress()` for long-running commands
- Escape HTML in webview content via `escapeHtml()`
